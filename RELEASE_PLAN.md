# Focus Flow Release Plan

## 目标

把发版从“一个功能打一次包”改成“批次开发、统一验证、统一打包”。这样我们可以减少你反复安装的成本，也能在每次交付前把数据安全、构建和桌面体验一起检查一遍。

## 发版节奏

- 小功能先进入当前开发批次，不单独打包。
- 每个批次建议包含 3-5 个低风险体验优化，或 1 个高风险基础能力改动。
- 只有在批次完成后，才运行完整发版验证并生成新的 DMG。
- 如果遇到数据丢失风险、构建阻塞、无法启动 App 这类问题，可以临时做 patch 包。

## 版本命名

- `0.1.x`: 桌面 App 稳定性、安装包、基础交互和数据安全修复。
- `0.2.x`: 核心任务流体验优化，包括快速录入、Today 主线、键盘操作和界面重构。
- `0.3.x`: Mac 集成，包括小组件数据快照、深链接、菜单栏/托盘等原生能力。

## 发版门槛

每次生成安装包前必须完成：

- `npm run lint`
- `npm run build`
- `npm run tauri:build`
- 启动打包后的 `.app` 做一次基础冒烟测试
- 检查用户数据文件没有被清空或错误迁移
- 确认 DMG 路径和版本号

## 当前批次：v0.1.1 桌面基础体验

目标：让当前桌面 App 更适合日常使用，同时减少安装摩擦。

- 完成“置顶/不置顶”按钮和图标。
- 验证 Tauri always-on-top 权限和打包后行为。
- 保留当前新版 logo 和 DMG 打包形式。
- 增加发布检查清单，避免再次出现存储切换导致的误解。
- 继续拆分首页组件，但只做低风险拆分，不改变数据结构。

建议打包时机：以上项目全部完成后统一打一个 `0.1.1` DMG。

## 下一批次：v0.2.0 主路径效率

目标：把每天最常用的路径打磨顺，不急着堆复杂功能。

- 把快速录入拆成独立组件，并支持更明确的添加反馈。
- 把 Today 主线拆成独立组件，强化 1-3 个主任务的视觉层级。
- 增加更多键盘快捷键，例如快速聚焦输入框、完成当前任务、打开工具箱。
- 增加备份/恢复入口，给桌面数据更强的安全感。状态：已开始，当前支持手动磁盘备份、复制数据位置、导入/重置前自动备份、自定义数据目录和恢复默认目录。
- 调整工作台信息密度，让 Batch、Review、日报、项目管理默认更安静。

建议打包时机：主路径 UI 和备份/恢复入口完成后统一打一个 `0.2.0` DMG。

## 后续批次：v0.3.0 Mac 集成

目标：开始接入 macOS 原生体验，但先用小步验证。

- 为小组件准备只读数据快照，例如 Today 任务、待处理数量和当前焦点任务。状态：数据桥已开始，当前写出 `focus-flow-widget-snapshot.json`。
- 评估 WidgetKit 扩展和 App Group 数据共享方案。
- 增加深链接草案，例如 `focusflow://today` 和 `focusflow://capture`。
- 评估菜单栏、全局快捷键、快速录入浮窗的优先级。

建议打包时机：先做小组件数据快照和深链接原型，验证可行后再进入 WidgetKit 原生扩展。

## 不打包的情况

- 只改内部组件结构，且用户体验没有明显变化。
- 只改文档或计划。
- 单个低风险按钮、文字或样式调整。
- 还没有跑完完整发版门槛。
